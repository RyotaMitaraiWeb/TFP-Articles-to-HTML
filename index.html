Convert The Flying Press articles to HTML code by Ryota Mitarai. GitHub repository

How to use

  1. Take the BBCode of the article and paste it in the form below
    • To take the BBCode, click "Reply" on the desired post and then copy the relevant part of the reply.
  2. Click the "Convert" button and copy the output from the next field
    • To copy the entire output, click into the form and press Ctrl + A or (if on Mac) Cmd + A
  3. Fix any errors that the tool may have committed and add anything that the tool couldn't add.
  4. To validate your article, you need to have a live preview from the SCMS (which you can only use if you are badged). Here's how to do it:
    • Open your live preview. There's a link "Preview" next to each HTML file, which you can click to open the preview
    • Access the code of the whole page through "View Source". You can right-click and then select "View page source" or similar or press Ctrl + U or (on Mac) Option + Cmd + U on most browsers.
    • Copy the entire code of the source
    • Open the W3 Validator (validate by direct input) and paste your code. In most cases, that tool should detect the DOCTYPE, but if it's not enabled, you can enable it through "More Options".
    • Check the errors and see what you can fix. If the only errors are ones that you cannot fix (as in, you don't have access to this part of the code), then you are good.
    • If there are errors that you can fix, fix them in a descending order and fix them one at a time. Some errors are caused directly by another, so fix the first ones so that later ones get fixed as well.
  5. If you have a sample team in the article, copy that separately and input it in the text field of the next section.
    • Convert each sample team separately (e.g. if there are three, convert them one by one).
  6. For more information on how the tool works, check out the GitHub repository, where you can find a detailed documentation.
